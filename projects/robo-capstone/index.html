Project Full Send

FPV Racing Simulator for the Redcat Racing Volcano EPX

Project Full Send is a real-world RC racing simulator that combines a physical RC vehicle, a full driver interface, and a live first-person video feed into one immersive system. The goal is to let a user drive the vehicle using a steering wheel and custom pedal inputs while viewing the track from the car’s onboard perspective.

The project blends robotics, embedded control, simulator design, RF video transmission, and mechanical integration. Instead of relying on expensive commercial simulator hardware, the system is built around modular, accessible components and a custom simulator rig that can continue evolving over time.

My work on the project includes research into GPS integration, early Python interface development, electronics sourcing, and system planning as part of the larger team effort to connect the driver controls, vehicle platform, and FPV pipeline into one usable prototype.

What the System Does

Immersive control interface paired with a real RC platform

The finished simulator is intended to let a user control a Redcat Racing Volcano EPX using a steering wheel and custom gas and brake pedals rather than a handheld RC transmitter. A live 5.8GHz FPV camera mounted on the vehicle streams video to a headset or monitor so the user drives from the vehicle’s point of view.

The broader vision also includes optional telemetry, lap timing, and coordinate-based mapping so the simulator becomes more than a basic RC setup. It becomes a platform for control-system experimentation, driver feedback, and future expansion into more advanced remote-operation concepts.

  • Real RC vehicle platform used as the physical base system
  • Steering wheel and pedal conversion for more realistic driving input
  • FPV video feed transmitted from the car to a display or headset
  • PWM-based steering and throttle signal generation
  • Potential HUD and telemetry features for lap timing and track awareness
  • Modular design intended for continued upgrades and testing

System Architecture

Driver controls, vehicle control electronics, video transmission, and optional telemetry working together

Core Hardware Layers

  • Vehicle platform: Redcat Racing Volcano EPX chassis, drivetrain, steering servo, ESC, and battery system
  • Driver interface: steering wheel, gas pedal, and brake pedal adapted into a simulator-style control station
  • FPV subsystem: 5.8GHz camera and receiver pipeline for low-latency first-person driving
  • Control electronics: Arduino and related controller hardware used to process inputs and generate steering / throttle outputs
  • Optional navigation layer: GPS, telemetry radios, and additional data systems for tracking and future expansion

How It Is Intended to Operate

Steering wheel and pedal inputs are read, normalized, and converted into usable control signals. Those signals are then mapped into proportional outputs for the steering servo and throttle system so the vehicle can respond more like a real driving interface instead of an all-or-nothing RC trigger.

In parallel, the onboard camera transmits a live view from the car. That feed is received and displayed to the driver through a headset or monitor, creating the simulator’s first-person perspective. Optional telemetry and GPS data can later support lap timing, position tracking, and richer feedback systems.

Engineering Focus Areas

The project is as much about integration and testing as it is about the final demo

Mechanical + User Interface

  • Building a custom simulator frame for the seat, steering wheel, pedal assembly, and monitor
  • Designing the rig to be stable, ergonomic, and usable for demonstrations
  • Evaluating pedal travel, steering range, and mounting positions for better control feel
  • Developing CAD-supported layouts for camera mounting and supporting structures

Control + Electronics

  • Mapping steering and throttle inputs into proportional PWM outputs
  • Improving throttle behavior to avoid abrupt 0–100% acceleration
  • Testing compatibility between RC electronics, controller hardware, and microcontrollers
  • Managing power distribution, wiring organization, and expansion paths for telemetry and sensors

Progress and Challenges

A realistic engineering project means testing, redesign, and fallback plans

Progress So Far

  • FPV feed successfully connected from vehicle to viewing hardware

Why This Project Stands Out

It combines multiple engineering disciplines into one demonstrable platform

This project is a strong portfolio piece because it is not just one isolated subsystem. It ties together embedded control, RC vehicle dynamics, simulator ergonomics, FPV video systems, structural design, hardware reuse, and future telemetry concepts into a single build. It also shows the ability to work within real constraints: donated components, evolving requirements, testing-driven decisions, and staged integration.

The result is a platform that has value beyond the class itself. It can serve as a robotics demonstration system, a controls-learning tool, a simulator prototype, and a base for future experiments involving GPS, path tracking, and remote operation concepts.
