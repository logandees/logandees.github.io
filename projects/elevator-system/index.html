4-Floor Arduino Elevator

A fully custom elevator system I designed, built, and coded from start to finish in a few days. The build uses an Arduino Mega, a NEMA 17 stepper-driven lead screw lift, floor call logic, in-car controls, and a servo-driven sliding door with hold-open behavior.

Arduino Mega NEMA 17 + Lead Screw Servo Door 4 Floors

Project Overview

This project was built to simulate a working four-floor elevator with realistic button-based control, automatic floor-to-floor motion, and door behavior. Rather than just making a static model, I focused on making the system behave like a real machine: floor requests are tracked, the lift travels to commanded floors, the door opens on arrival, a hold-open input extends the door timing, and the car returns to the home floor after inactivity.

The system combines CAD-designed structure, a vertical lift driven by a stepper motor and lead screw, an Arduino-based control system, and an actuator-driven door. It also includes both external floor call buttons and interior controls so the build feels like a complete elevator system instead of just a moving platform.

What it does

  • Serves 4 floors with floor call inputs
  • Uses a stepper motor + lead screw for vertical travel
  • Uses a servo-driven sliding door at each stop
  • Supports a door hold / obstruction input
  • Returns to Floor 1 when idle
  • Built around an Arduino Mega control system

Full system layout

The assembly view shows the overall structure of the project: the multi-floor tower, elevator car, support structure, and lower drive area. This is the cleanest way to introduce the whole design before getting into the electrical and control details.

Lift mechanism

The car rides on a screw-driven lift system powered by a NEMA 17 stepper motor. That setup gave me precise floor-to-floor positioning and made the motion easier to control in code than a looser drive approach.

Electrical control

The electrical side ties together the Arduino Mega, motor driver hardware, floor inputs, and the door control system. The project needed reliable signal flow between the buttons, the controller, and the actuators so the machine responded correctly every time a floor was requested.

Logic and code behavior

The software side is what made the build feel complete. The elevator waits for inputs, tracks requested floors, moves to the correct location, opens the door on arrival, responds to a hold-door input, and then returns home after the system is inactive.
