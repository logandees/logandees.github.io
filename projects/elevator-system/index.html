Elevator Design Project

A four-floor elevator prototype built around a 3D-printed structure, an Arduino MEGA control system, a NEMA 17 stepper-driven lift, and a servo-actuated sliding door. Project overview

The project goal was to build a working four-floor elevator that could be called from each floor, accept interior floor requests, open and close its door automatically, keep the door open when the hold button is pressed, and return to Floor 1 when idle. The final build used a stepper-driven lift, a servo-powered door system, and a mostly 3D-printed structure.

Control logic

The Arduino MEGA stores floor requests, moves the car to requested floors, opens the door at stops, and sends the car back to the ground floor after inactivity.

Mechanical system

The elevator car travels vertically on a lead-screw lift system, while the door mechanism uses servo-driven gears to create sliding motion at each stop.

Modular construction

The design separates the base, rails, floors, roof, carriage, and door components into distinct modeled parts, which makes the assembly easier to document and revise.

Documentation package

Design highlights

The writeup and drawings make it clear that this was built as both a working prototype and a documented controls project.

  • Four-floor call systemEach floor has a call input, and the top control area represents the interior floor controls.
  • Automatic door cycleThe door opens at floor arrival, stays open for the required time, and remains open while the hold input is active.
  • Nearest-stop behaviorThe motion logic prioritizes requested stops based on the elevator's current position and direction of travel.
  • Return-to-home behaviorWhen the system is not in use, the elevator returns to Floor 1 for the next request.
  • Future improvement pathPossible next steps include adding the planned buzzer and LED warning behavior, strengthening the frame, and refining the door system.

System behavior

These were the main behaviors and design requirements carried through the final project.

Floor requests

Exterior call buttons and interior floor buttons send requests to the controller so the elevator can travel to the selected floors.

Door hold function

A dedicated hold button simulates an obstruction response by keeping the door open while the input remains active.

Step-by-step positioning

The stepper system uses measured floor positions so the car can stop repeatedly at each floor location rather than just running continuously.

Physical demonstration

The final build favors reliable motion and visible operation over speed, making it a better classroom demonstration system.

Future work

The report pointed to a few clear improvements that would make a next version even stronger.

  • Door alarm and LED feedbackThe intended long-hold warning behavior was planned but not fully finished before demonstration day.
  • Frame reinforcementAdditional rear or side supports would reduce shaking and make the structure feel more rigid during travel.
  • Door redesign optionsA center-opening two-door arrangement would better match a full-scale elevator experience.
